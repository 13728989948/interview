什么是SMP,NUMA,MPP

目前的商用服务器大体可以分为三类:SMP,NUMA,MPP

SMP(Symmetric Multi-Processor):对称多处理器结构

NUMA(Non-Uniform Memory Access):非一致存储访问结构

MPP(Massive Parallel Processing):海量并行处理结构

SMP(Symmetric Multi-Processor)

所谓对称多处理器结构，是指服务器中多个CPU对称工作，无主次或从属关系

系统中所有资源(CPU、内存、I/O等)都是共享的

每个CPU必须通过相同的内存总线访问相同的内存资源，因此随着CPU数量的增加，内存访问冲突将迅速增加，最终会造成CPU资源的浪费

NUMA(Non-Uniform Memory Access)

具有多个CPU模块，每个CPU模块由多个CPU(如4个)组成

每个CPU模块具有独立的本地内存、I/O槽口

每个CPU模块之间可以通过互联模块进行连接和信息交互

可以较好地解决原来SMP系统的扩展问题，在一个物理服务器内可以支持上百个CPU，访问本地内存的速度将远远高于访问远地内存

由于访问远地内存的延时远远超过本地内存，因此当CPU数量增加时，系统性能无法线性增加

NUMA架构适用于OLTP(On-Line Transaction Processing联机事务处理过程)事务处理环境

OLTP 系统强调数据库内存效率，强调内存各种指标的命令率，强调绑定变量，强调并发操作

MPP(Massive Parallel Processing)

由多个SMP服务器通过一定的节点互联网络进行连接，扩展性最好

每个节点只访问自己的本地资源(内存、存储)

MPP架构适用于OLAP(On-Line Analytical Processing联机分析处理)

OLAP 系统则强调数据分析，强调SQL执行市场，强调磁盘I/O，强调分区等
